## 2. 애플리케이션 실행

### 실행 명령어

프로젝트 루트 디렉토리(`C:\Reviewer\poc`)에서 아래 명령어를 실행합니다.

```powershell
.\gradlew.bat run --console=plain
```

> `--console=plain` 옵션은 Gradle의 진행 표시가 입력창을 가리지 않도록 하기 위해 반드시 붙여주세요.

### 최초 실행 시 동작

- 인터넷에서 라이브러리(Jackson)를 자동으로 다운로드합니다 (약 1~2분 소요).
- `data/users.json` 파일이 없으면 자동으로 빈 파일을 생성합니다.

### 정상 실행 화면

```
데이터 파일: data/users.json
══════════════════════════════════════
   사용자 관리 시스템  (JSON CRUD)
══════════════════════════════════════
 1. 전체 목록 조회
 2. ID로 검색
 3. 이름으로 검색
 4. 새 사용자 등록
 5. 사용자 정보 수정
 6. 사용자 삭제
 0. 종료
══════════════════════════════════════
메뉴 선택 >
```

### TC-01 새 사용자 등록 (Create)

**목적:** 새로운 사용자 데이터를 입력하고 JSON 파일에 저장되는지 확인합니다.

#### 테스트 1-1: 정상 등록

**메뉴에서 `4` 입력 → Enter**

아래 순서대로 값을 입력합니다.

| 입력 항목 | 입력값 |
|-----------|--------|
| 이름 | `홍길동` |
| 이메일 | `hong@example.com` |
| 나이 | `30` |
| 주소 - 도로명 | `테헤란로 123` |
| 주소 - 도시 | `서울` |
| 주소 - 우편번호 | `06234` |
| 전화번호 | `010-1234-5678` |

**입력 화면 예시:**
```
[새 사용자 등록]
이름 > 홍길동
이메일 > hong@example.com
나이 > 30
주소 - 도로명 > 테헤란로 123
주소 - 도시 > 서울
주소 - 우편번호 > 06234
전화번호 (쉼표로 구분, 예: 010-1234-5678,010-9999-0000) > 010-1234-5678
```

**기대 결과:**
```
등록 완료! (부여된 ID: 1)
```

> ID는 자동으로 부여됩니다. 첫 번째 등록 사용자의 ID는 항상 `1`입니다.

#### 테스트 1-2: 전화번호 여러 개 등록

전화번호 항목에 쉼표로 구분해 복수 입력합니다.

```
전화번호 > 010-1111-2222,010-3333-4444
```

**기대 결과:** 두 개의 전화번호가 모두 저장됩니다.

#### 테스트 1-3: 사용자 추가 등록 (ID 자동 증가 확인)

`4`번 메뉴를 한 번 더 선택해 두 번째 사용자를 등록합니다.

| 입력 항목 | 입력값 |
|-----------|--------|
| 이름 | `김영희` |
| 이메일 | `kim@example.com` |
| 나이 | `25` |
| 주소 - 도로명 | `강남대로 2` |
| 주소 - 도시 | `서울` |
| 주소 - 우편번호 | `06101` |
| 전화번호 | `010-9999-8888` |

**기대 결과:**
```
등록 완료! (부여된 ID: 2)
```

> ID가 1 → 2로 자동 증가했음을 확인합니다.

│ 이름     : 홍길동

### TC-05 사용자 정보 수정 (Update)

**목적:** 특정 필드를 수정했을 때 해당 값만 변경되고 나머지는 유지되는지 확인합니다.

**메뉴에서 `5` 입력 → Enter**

#### 테스트 5-1: 이메일 수정

```
수정할 사용자 ID > 1
```

현재 정보가 출력된 뒤 수정할 필드 목록이 표시됩니다.

```
수정할 필드를 선택하세요:
  1. 이름       현재: 홍길동
  2. 이메일     현재: hong@example.com
  3. 나이       현재: 30
  4. 도로명     현재: 테헤란로 123
  5. 도시       현재: 서울
  6. 우편번호   현재: 06234
  7. 전화번호   현재: [010-1234-5678]
필드 선택 > 2
새 값 > updated@example.com
```

**기대 결과:**
```
수정 완료!
```

**검증:** `2`번 메뉴(ID 검색)에서 ID `1`을 검색해 이메일이 `updated@example.com`으로 바뀌었는지, 이름·나이·주소 등 다른 필드는 그대로인지 확인합니다.

#### 테스트 5-2: 나이 수정

```
수정할 사용자 ID > 1
필드 선택 > 3
새 값 > 35
```

**기대 결과:** 나이가 `35`로 변경됩니다.

#### 테스트 5-3: 전화번호 수정 (복수 입력)

```
수정할 사용자 ID > 1
필드 선택 > 7
새 값 > 010-5555-5555,010-6666-6666
```

**기대 결과:** 전화번호가 두 개로 변경됩니다.

## 5. JSON 파일 직접 확인

테스트 중 데이터가 파일에 올바르게 저장되고 있는지 직접 확인할 수 있습니다.

### 파일 위치

```
C:\Reviewer\poc\data\users.json
```

### 파일 내용 예시 (사용자 1명 등록 후)

```json
[ {
  "id" : 1,
  "name" : "홍길동",
  "email" : "hong@example.com",
  "age" : 30,
  "address" : {
    "street" : "테헤란로 123",
    "city" : "서울",
    "zip_code" : "06234"
  },
  "phone_numbers" : [ "010-1234-5678" ]
} ]
```

### 파일로 검증해야 할 항목

| 동작 | 파일에서 확인할 내용 |
|------|---------------------|
| Create 후 | 배열에 새 객체가 추가되었는지 |
| Update 후 | 해당 ID 객체의 변경 필드만 수정되었는지 |
| Delete 후 | 해당 ID 객체가 배열에서 제거되었는지 |
| 앱 재시작 후 | 파일 내용이 유지되는지 |

> 메모장, VS Code, IntelliJ 등 어떤 텍스트 편집기로도 확인 가능합니다.

## 6. 전체 흐름 통합 테스트

**목적:** Create → Read → Update → Delete를 순서대로 수행해 전체 흐름이 일관성 있게 동작하는지 검증합니다.

### 단계별 실행 순서

```
Step 1. 애플리케이션 실행
        .\gradlew.bat run --console=plain

Step 2. [Create] 메뉴 4 → 이순신 / lee@example.com / 45 / 충무로 1 / 서울 / 04522 / 010-7777-7777 입력
        → "등록 완료! (부여된 ID: 1)" 확인

Step 3. [Read - 전체] 메뉴 1
        → ID 1, 이름 이순신 출력 확인

Step 4. [Read - ID] 메뉴 2 → ID: 1
        → 이순신 정보 출력 확인

Step 5. [Read - 이름] 메뉴 3 → 키워드: 이
        → 이순신 포함 결과 확인

Step 6. [Update] 메뉴 5 → ID: 1 → 필드: 2 (이메일) → 새 값: new@example.com
        → "수정 완료!" 확인

Step 7. [Read - ID] 메뉴 2 → ID: 1
        → 이메일이 new@example.com 으로 변경된 것 확인
        → 이름·나이 등 나머지 필드는 그대로인 것 확인

Step 8. [Delete] 메뉴 6 → ID: 1 → y
        → "삭제 완료!" 확인

Step 9. [Read - 전체] 메뉴 1
        → "등록된 사용자가 없습니다." 확인

Step 10. 메뉴 0 → 종료
```

모든 단계에서 기대 결과와 일치하면 통합 테스트 통과입니다.

### Create
- [ ] 사용자 등록 시 ID가 자동 부여된다
- [ ] 두 번째 등록 시 ID가 1 증가한다
- [ ] 전화번호를 쉼표로 구분해 복수 등록된다
- [ ] 등록 후 `data/users.json`에 데이터가 추가된다

### Read
- [ ] 전체 목록에 등록된 모든 사용자가 출력된다
- [ ] 데이터가 없을 때 "등록된 사용자가 없습니다." 메시지가 출력된다
- [ ] ID 검색으로 특정 사용자를 정확히 조회할 수 있다
- [ ] 존재하지 않는 ID 검색 시 안내 메시지가 출력된다
- [ ] 이름 부분 일치 검색이 동작한다
- [ ] 존재하지 않는 이름 검색 시 안내 메시지가 출력된다

### Update
- [ ] 수정한 필드만 변경되고 나머지는 유지된다
- [ ] 수정 후 `data/users.json`에 변경 내용이 반영된다
- [ ] 존재하지 않는 ID 수정 시 안내 메시지가 출력된다
- [ ] 잘못된 필드 번호 입력 시 데이터 변경 없이 메뉴로 복귀한다

### Delete
- [ ] `y` 입력 시 데이터가 삭제된다
- [ ] `n` 입력 시 데이터가 삭제되지 않는다
- [ ] 삭제 후 `data/users.json`에서 해당 데이터가 제거된다
- [ ] 존재하지 않는 ID 삭제 시 안내 메시지가 출력된다

### 예외 처리
- [ ] ID 입력란에 문자 입력 시 오류 메시지 출력 후 정상 동작한다
- [ ] 나이 입력란에 문자 입력 시 오류 메시지 출력 후 정상 동작한다
- [ ] 잘못된 메뉴 번호 입력 시 오류 메시지 출력 후 메뉴가 재표시된다

### 데이터 영속성
- [ ] 앱 종료 후 재실행해도 데이터가 유지된다
- [ ] `data/users.json` 파일이 사람이 읽기 좋은 형태(pretty print)로 저장된다
